# Publishing accessible documents
\b0\fs29\fsmilli14667 \cf3 \outl0\strokewidth0 \strokec3 \
\pard\pardeftab720\sl400\partightenfactor0
\cf3 \
Documents published on GOV.UK or other [public sector websites must meet accessibility standards] (link to overview guide). This is so they can be used by as many people as possible, including those with disabilities.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 If your document does not meet the standards, you could be breaking the law.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\b\fs29\fsmilli14667 \cf3 ## What to do as a document author
\f1\b0\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Follow these steps when you write a document. If you have questions, contact your website publishing team.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\b\fs29\fsmilli14667 \cf3 ### 1. Think about format
\f1\b0\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Wherever possible, publish in HTML. It\'92s the best way to reach as many people as possible.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Documents like PDFs make your content [harder to find, use and maintain](https://gds.blog.gov.uk/2018/07/16/why-gov-uk-content-should-be-published-in-html-and-not-pdf/). And it can be difficult for users to customise them for ease of reading, and often they do not work very well with assistive technologies like screen readers.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Contact your website team to find out more about publishing in HTML.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 If you do need to publish a document, it should be in addition to an HTML version.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\b\fs29\fsmilli14667 \cf3 ### 2. Keep the language simple
\f1\b0\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Write in language that\'92s as simple as possible.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Simple language makes your document accessible to people with cognitive impairments and learning disabilities.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 And research shows that everyone prefers simple language, including specialist audiences. Because it allows them to understand information as quickly as possible.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Where you need to use technical terms, abbreviations or acronyms, explain what they mean the first time you use them.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\b\fs29\fsmilli14667 \cf3 ### 3. Keep the document simple
\f1\b0\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Keep sentences and paragraphs short. 
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Use a sans serif font like Arial or Helvetica. Use a minimum size of 12 points.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Use sentence case. Avoid all caps text and italics.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Make sure the text is left aligned, not justified.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Avoid underlining, except for links.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Documents with single, continuous columns of text are easier to make accessible than documents with a more complex layout.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Only use tables for data. Keep tables simple: avoid splitting or merging cells.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Do not use colour alone to get across meaning.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 If you\'92re using images, think about how you\'92ll make the content accessible to people with a visual impairment. Two options are:
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 - make the same point in the text of the document (so people with visual impairments get the information they need - the image is there as an extra for people who are able to see it)
\f1\fs24 \

\f0\fs29\fsmilli14667 - give the person converting or uploading the document for you [alt text (\'91alternative text\'92)]({\field{\*\fldinst{HYPERLINK "https://webaim.org/techniques/alttext/"}}{\fldrslt \cf4 \ul \ulc4 \strokec4 https://webaim.org/techniques/alttext/}}) for the image
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Avoid footnotes where possible. Provide explanations inline instead.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\b\fs29\fsmilli14667 \cf3 ### 3. Give the document a structure
\f1\b0\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Break up your document to make it more readable. Use bullet points, numbered steps and subheadings.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 Do not use bold to mark up subheadings. Use styles to create a hierarchy of headings: \'91heading 1\'92, \'91heading 2\'92 and so on. Ask your website publishing team if you\'92re not sure how to do this.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\b\fs29\fsmilli14667 \cf3 ####If you\'92re using Windows
\f1\b0\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 [Non Visual Desktop Access (NVDA)](http://www.nvda-project.org/) is a free open source screen reader for Windows. It can be installed to the desktop or run from a portable USB drive.
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 With NVDA running, open the PDF and use the following commands to check the PDF:
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 - from the top of the PDF (with the numlock off), use Numpad 0 + Numpad 2 to read the PDF from top to bottom and check the reading order
\f1\fs24 \

\f0\fs29\fsmilli14667 - use the tab key to move through the PDF and check the tab order
\f1\fs24 \

\f0\fs29\fsmilli14667 - use the h key to move through the PDF and check the heading structure
\f1\fs24 \

\f0\fs29\fsmilli14667 - use the g key to move through the PDF and check for text descriptions
\f1\fs24 \
\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0

\f0\fs29\fsmilli14667 \cf3 These commands will also work with the Jaws screen reader from Freedom Scientific.

\pard\pardeftab720\sl280\partightenfactor0
\cf3 \
\pard\pardeftab720\sl400\partightenfactor0
